# 🔒 端到端加密点对点聊天系统

全程加密、无日志、不监控的浏览器网页端一对一加密聊天。纯网页访问，无需安装客户端，核心代码不到 600 行。

## ✨ 特性

- **端到端加密 (E2EE)**：ECDH 密钥协商 + xsalsa20-poly1305 对称加密，聊天内容仅双方可解密
- **点对点直连通信**：基于 WebRTC，文本消息点对点直传，不经过服务器中转
- **零信任架构**：信令服务器仅转发连接握手信息，无法解密任何聊天内容
- **链接即授权**：发起人生成加密分享链接，被访客打开即建立加密通道
- **多会话隔离**：多访客同时连接发起人，通道独立、密钥独立，访客间互不可见
- **零持久化存储**：无账号、无聊天记录、无用户数据，页面关闭即清除
- **暗色主题 UI**：居中卡片式布局，清晰的安全状态指示

## 🚀 快速开始

```bash
# 克隆仓库
git clone https://github.com/sunqichaoPHP/secure-p2p-chat.git
cd secure-p2p-chat

# 安装依赖
npm install

# 启动服务
npm start
```

打开 `http://localhost:3000` 即可使用。

## 📖 使用方式

### 发起人（创建会话）

1. 打开页面，点击 **「开启加密聊天服务」**
2. 自定义昵称
3. 点击 **「复制」** 获取加密分享链接
4. 将链接发送给要聊天的人

### 访客（加入会话）

1. 打开收到的加密链接
2. 输入昵称，点击 **「加入加密通道」**
3. 等待加密通道建立，开始聊天

### 关闭会话

发起人关闭页面或点击停止服务 → 链接立即失效，所有通道断开。

## 🔐 安全机制

| 层级 | 措施 | 说明 |
|------|------|------|
| 密钥交换 | ECDH (Curve25519) | 公私钥仅存浏览器内存，不上传 |
| 消息加密 | xsalsa20-poly1305 | tweetnacl secretbox，对称加密 |
| 传输层 | WebRTC 点对点 | 聊天数据直连传输，不经过业务服务器 |
| 信令层 | Socket.io | 仅转发 SDP/ICE 握手信息，不触碰聊天内容 |
| 授权 | 一次性授权码 | 嵌入 URL 哈希片段（#），浏览器本地读取，不发送到服务器 |
| 存储 | 零持久化 | 无聊天日志、无用户身份、关闭即清除 |

### 信令服务器职责边界

```
✅ 转发的：房间管理、WebRTC 信令 (SDP/ICE)、公钥交换
❌ 不做的：存储聊天消息、解密内容、记录用户身份、解析聊天内容
```

## 🛠️ 技术栈

| 层级 | 技术 |
|------|------|
| 点对点通信 | WebRTC (RTCPeerConnection + DataChannel) |
| 加密 | tweetnacl.js (ECDH + secretbox) |
| 信令 | Node.js + Express + Socket.io |
| 前端 | 原生 HTML/CSS/JS，零框架依赖 |

## 🔄 数据流图

```
发起人浏览器                          访客浏览器
    │                                    │
    ├─ 生成 ECDH 密钥对                  ├─ 从链接读取发起人公钥
    ├─ 创建房间 (socket.io)              ├─ 生成自己的密钥对
    │                                    │
    │  ←———— 信令(公钥/ICE) ————→      │
    │       (仅握手，服务器不解密)         │
    │                                    │
    │  ←==== WebRTC 点对点直连 =====→     │
    │       (密文传输，不走服务器)         │
    │                                    │
    ▼                                    ▼
  明文 ←→ AES 解密/加密 ←→ 密文        明文 ←→ AES 解密/加密 ←→ 密文
```

## 🌐 部署

前端为纯静态文件+信令服务为 Node.js 应用，生产环境需配置 HTTPS（WebRTC 强制要求）。

```bash
# 用 nginx 反向代理 + Let's Encrypt
# 或部署到支持 HTTPS 的云服务
PORT=443 npm start
```

最低配置要求：
- Node.js 16+
- HTTPS 证书 (Let's Encrypt 免费)
- 1 核 512MB 即可运行
